# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in İletiniz PHP SDK, please **do not open a public GitHub issue**.

Instead, send your security reports to: **security@iletiniz.com**

Please include in your report:
- Description and impact of the vulnerability
- Steps to reproduce
- Affected version(s)
- A suggested fix, if possible

## Response Time

We acknowledge all security reports within **72 hours**. After validating the vulnerability, we share a fix plan and an estimated release timeline.

Once a fix is released, with your permission we credit your report in the CHANGELOG.

## Supported Versions

| Version | Support Status |
|---|---|
| 0.x | ✅ Active security support |
| Older | ❌ No support — please upgrade |

## Disclosure Policy

We follow a coordinated disclosure approach. Please do not publicly disclose the vulnerability until a fix has been released.
